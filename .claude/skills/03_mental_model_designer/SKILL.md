# Mental Model Designer

## Purpose

Turn a technical idea into one durable intuition.

## Responsibilities

- Choose exactly one primary mental model.
- Map concept parts to familiar objects.
- Clarify what the analogy explains.
- Clarify where the analogy breaks.
- Produce a one-sentence memory hook.

## Inputs

- Lesson topic
- Story opening
- Target audience
- Core technical difficulty

## Outputs

- Primary mental model
- Mapping table
- Analogy limits
- Memory hook

## Procedure

1. Identify the hardest conceptual leap.
2. Find one ordinary-world mechanism that preserves the same causal structure.
3. Map each important AI engineering symbol or process to the analogy.
4. State the analogy's boundary conditions.
5. Compress the idea into one memorable sentence.

## Examples

- Roofline model -> chef waiting on a conveyor belt
- PagedAttention -> hotel rooms handed out one at a time instead of reserved floors
- Actor-critic -> student plus teacher
- Speculative decoding -> junior drafts, senior signs off

## Failure Cases

- Multiple metaphors competing in one lesson.
- An analogy that is catchy but structurally wrong.
- A metaphor that explains the intuition but hides the update rule.
- No limits stated for the analogy.

## Checklist

- [ ] Exactly one primary mental model is chosen.
- [ ] The mapping is explicit.
- [ ] The analogy is structurally faithful.
- [ ] The limits are stated.
- [ ] The memory hook is short.
